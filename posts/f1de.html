1.初始MyBatis

1.1什么是MyBatis

  • mybatis是支持普通SQL查询、存储过程和高级映射的优秀持久层框架。

  • 它消除了几乎所有的JDBC代码和参数的手工设置以及结果集的检索

  • 使用简单的XML或注解用于配置和原始映射，将接口和Java的POJO（Plain Ordinary Java Objects，普通的Java对象）映射成数据库中的记录。

1.2什么是持久化

数据持久化

  • 持久化就是将程序的数据在瞬时状态向持久状态转化的过程
  • 内存：断电即失
  • 持久化：数据库，io文件

2.2创建模块

  • 编写mybatis的核心配置文件

    <!--核心配置文件-->
<configuration>
    <!--环境使用的是development，可以定义多个环境-->
    <environments default="development">
        <environment id="development">
            <!--事务管理，类型是JDBC-->
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <property name="driver" value="com.mysql.jdbc.Driver"/>
                <property name="url" value="jdbc:mysql://localhost:3306/mybatis?useSSL=true&amp;useUnicode=true&amp;characterEncoding=UTF-8"/>
                <property name="username" value="root"/>
                <property name="password" value="123456"/>
            </dataSource>
        </environment>
    </environments>
</configuration>

<!--&amp;是&的转义字符-->

  • 编写mybatis工具类

//获取SqlSession对象
public class MyBatisUtils {
    private static SqlSessionFactory sqlSessionFactory;
    static {
        String resource = "mybatis-config.xml";
        try {
            InputStream inputStream = Resources.getResourceAsStream(resource);
            SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }

    public static SqlSession getSqlSession(){
        return sqlSessionFactory.openSession();
    }
}

2.3编写代码

  • 实体类

    package cn.yunmo.pojo;

//实体类
public class User {
    private int id;
    private String name;
    private String pwd;

    public User() {

    }

    public User(int id, String name, String pwd) {
        this.id = id;
        this.name = name;
        this.pwd = pwd;
    }

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public String getPwd() {
        return pwd;
    }

    public void setPwd(String pwd) {
        this.pwd = pwd;
    }

    @Override
    public String toString() {
        return "User{" +
                "id=" + id +
                ", name='" + name + '\'' +
                ", pwd='" + pwd + '\'' +
                '}';
    }
}

  • Dao接口

    package cn.yunmo.dao;

import cn.yunmo.pojo.User;

import java.util.List;

public interface UserDao {
    //获取所有的user
    List<User> getUserList();
}

  • 接口实现类

    由原来的UserDaoImpl转换为一个Mapper配置文件

    <?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<!--namespace=绑定一个对应的mapper接口-->
<!--将这个xml文件与UserDao绑定了,类似于之前的实现了UserDao这个接口-->
<mapper namespace="cn.yunmo.dao.UserDao">
    <!--类似于之前的实现接口中的方法-->
    <!--
        id:即方法名
        resultType:即执行这个sql后的返回结果类型
    -->
    <select id="getUserList" resultType="cn.yunmo.pojo.User">
        select * from user
    </select>
</mapper>

2.4测试

注意：

1. org.apache.ibatis.binding.BindingException: Type interface cn.yunmo.dao.UserDao is not known to the MapperRegistry

绑定异常，每一个Mapper.xml都需要在MyBatis核心配置文件中注册

<!--每一个Mapper.xml都需要早MyBatis核心配置文件中注册-->
    <mappers>
        <mapper resource="cn/yunmo/dao/UserMapper.xml"></mapper>
    </mappers>

2. java.lang.ExceptionInInitializerError

maven默认只将resources中的资源文件打包，解决办法是将如下代码放入POM.xml文件中

<!--防止我们资源导出失败-->
<build>
        <resources>
            <resource>
                <directory>src/main/java</directory>
                <includes>
                    <include>**/*.xml</include>
                    <include>**/*.properties</include>
                </includes>
                <filtering>true</filtering>
            </resource>
            <resource>
                <directory>src/main/resources</directory>
                <includes>
                    <include>**/*.xml</include>
                    <include>**/*.properties</include>
                </includes>
            </resource>
        </resources>
    </build>

3.CRUD

  1. namespace

namespace中的包名要和Dao/mapper接口的包名一致

  1. mapper中sql语句中参数
  • id：就是对应的namespace中的方法名
  • resultType：sql语句执行的返回值
  • parameterTyper：参数类型

  1. 步骤

    • 编写接口中的方法
    • 在对应的mapper.xml配置文件中编写对应的sql语句
    • 测试

  2. select

    编写接口中的方法

    //获取所有的user
List<User> getUserList();

//根据id查询用户
User getUserById(int id);

    编写mapper中的sql语句

    <!--查询所有的用户-->
    <select id="getUserList" resultType="cn.yunmo.pojo.User">
        select * from user
    </select>

    <!--根据id用户-->
    <select id="getUserById" parameterType="int" resultType="cn.yunmo.pojo.User">
        select * from user where id= #{id}
        <!--方法传的参数名叫id-->
    </select>

    编写测试方法

4.万能Map

假设实体类，或者数据库中的表，字段或者参数过多，当我们修改数据时用对象需要传递所有参数，此时我们可以考虑使用Map

接口中定义方法

int addUser(Map<String,Object> map);

mapper.xml配置文件

<!--传递Map中的key-->
<insert id="addUser" parameterType="map">
    insert into user(id,pwd) values(#{userId},#{password});
</insert>

测试方法

@Test
public void addeUser(){
    SqlSession sqlSession = MyBatisUtils.getSqlSession();
    UserMapper mapper = sqlSession.getMapper(UserMapper.class);

    Map map = new HashMap<String,Object>();
    map.put("userId",1);
    map.put("password","888888");
    int num = mapper.addUser(map);
    if(num>0){
        System.out.println("删除成功!");
    }

    //提交事务
    sqlSession.commit();

    sqlSession.close();
}

map传递参数，直接在sql中取出key即可

对象传递参数，直接在sql中取出对象的属性

只有一个基本类型的参数，可以直接在sql中取得

多个参数用Map

5.模糊查询

接口方法

List<User> getUserLike(String value);

mapper查询语句

<select id="getUserLike" resultType="cn.yunmo.pojo.User"">
        select * from user where name like #{value};
</select>

测试

@Test
public void addeUser(){
    SqlSession sqlSession = MyBatisUtils.getSqlSession();
    UserMapper mapper = sqlSession.getMapper(UserMapper.class);
    List<User> userList = mapper.getUserLike("%杨%");
    for(User user:userList){
        System.out.println(user);
    }
    //提交事务
    sqlSession.commit();

    sqlSession.close();
}
