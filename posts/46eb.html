1.简介

Spring框架是由于软件开发的复杂性而创建的轻量级控制反转]（IoC）和面向切面（AOP）的容器框架。

  • 目的：解决企业应用开发的复杂性
  • 功能：使用基本的JavaBean代替EJB，并提供了更多的企业应用功能
  • 范围：任何Java应用
  • 优点：
    • 开源的免费的容器(框架)
    • 轻量级的，非入侵式的框架
    • 支持事务的处理，对框架整合的支持
  • 特性：
    • 控制反转(IOC)
    • 面向切面编程(AOP)

Spring是一个轻量级的控制反转，面向切面编程的框架

3.2IOC本质

控制反转，是一种设计思想，DI(依赖注入)是实现IoC的一种方法，也有人认为DI只是IoC的另一种说法。没有IoC的程序中，我们使用面向对象编程，对象的创建与对象间的依赖关系完全硬编码在程序中，对象的创建由程序自己控制，控制反转后将对象的创建转移给第三方，所谓控制反转就是：获得依赖对象的方式反转了

控制反转是一种通过描述(xml或注解)并通过第三方去生产或获取特定对象的方式。在spring中实现控制反转的是Ioc容器，其实现方法是依赖注入

9.1.3.1byName自动装配

byName：会自动在容器上下文中查找和自己对象的set方法后面的值【setXXX，后面的值说的是XXX】对应的bean的Id进行注入

<bean id="cat" class="cn.yunmo.pojo.Cat"/>
<bean id="dog" class="cn.yunmo.pojo.Dog"/>

<bean id="people" class="cn.yunmo.pojo.People" autowire="byName">
        <property name="name" value="yangqi"/>
</bean>
9.1.3.2byType自动装配

byType：会自动在容器上下文中查找和自己对象属性类型相同【根据class指定，如果有多个bean的class值一样则不能装配】的bean,可以不要id

<bean class="cn.yunmo.pojo.People" autowire="byName">
       <property name="name" value="yangqi"/>
</bean>

注意：

  • byName的时候，需要保证所有的bean的id唯一，并且这个bean需要和自动注入的属性的set方法的值一致
  • byType的时候，需要保证所有的bean的class唯一，并且这个bean需要和自动注入的属性的类型一致

10.2@AutoWird

直接在属性上使用，也可以在set方法上使用

可以不要set方法，前提是你这个自动装配的属性在IoC(Spring)容器中存在

package cn.yunmo.pojo;

import org.springframework.beans.factory.annotation.Autowired;

public class People {
    private String name;
    @Autowired
    private Cat cat;
    @Autowired
    private Dog dog;

    @Override
    public String toString() {
        return "People{" +
                "name='" + name + '\'' +
                ", cat=" + cat +
                ", dog=" + dog +
                '}';
    }
}
<bean id="cat" class="cn.yunmo.pojo.Cat"/>
<bean id="dog" class="cn.yunmo.pojo.Dog"/>
<bean id="people" class="cn.yunmo.pojo.People"/>

科普

@Autowired(required = false)
  • 如果显示的定义AutoWird的required属性为false，说明这个对象可以为null，否则不允许为空

10.4@Qualifier

如果@Autowid自动装配的环境比较复杂，自动装配无法通过一个【@Autowid】完成的时候，我们可以使用@Qualifier(value=”dog2”)去配合@Autowid的使用，指定一个唯一的bean对象注入

@Autowired
@Qualifier(value="cat2")
    private Cat cat;

10.5@Resource

@Resource会先根据id去找，如果id与属性名相同则装配，否则再根据class找，如果是唯一的则装配，否则装配失败

当装配环境较复杂时，我们可以给@Resource指定一个bean

@Resource(name="cat2")
    private Cat cat;
<bean id="cat1" class="cn.yunmo.pojo.Cat"/>
<bean id="cat2" class="cn.yunmo.pojo.Cat"/>
<bean id="dog" class="cn.yunmo.pojo.Dog"/>

10.6@Resource和@Autowird的区别

  • 都是用来自动装配的，都可以放在属性字段上
  • @Autowird通过byName的方式实现的
  • @Resource默认通过byName的方式实现的，如果找不到名字，则通过byType的方式，都找不到的情况才报错

11.5小结

xml与注解：

  • xml更加万能，适合于任何场合，维护简单方便

package cn.yunmo.demo04;

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

//我们会用这个类自动生成代理类
public class ProxyInvocationHandler implements InvocationHandler {
    //被代理的接口(可以自己更换)
    private Object target;

    public void setTarget(Object target) {
        this.target = target;
    }

    //生成得到代理类实例
    public Object getProxy(){
        return Proxy.newProxyInstance(this.getClass().getClassLoader(),target.getClass().getInterfaces(),this);
    }

    //处理代理实例并返回结果
    //代理方法, 每次调用目标方法时都会进到这里
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        log(method.getName());
        //动态代理的本质就是使用反射机制
        Object result = method.invoke(target, args);
        return result;
    }
    public void log(String msg){
        System.out.println("执行了"+msg+"方法");
    }
}

     * @param method 目标对象的方法
     * @param objects 目标对象方法的参数列表，参数值列表

14AOP实现

14.1什么是AOP

面向切面编程（也叫面向方面），可以通过预编译方式和运行期动态代理实现在不修改源代码的情况下给程序动态统一添加功能的一种技术。AOP实际是GoF设计模式的延续，设计模式孜孜不倦追求的是调用者和被调用者之间的解耦，AOP可以说也是这种目标的一种实现。

16.Spring声明式事务

事务：要么都成功，要么都失败

16.1事务的原则

ACID原则：

  • 原子性
  • 一致性
  • 隔离性
    • 多个业务可能操作同一个资源，防止数据损坏
  • 持久性
    • 事务一旦提交，无论系统发生什么问题，结果都不会再被影响，被持久的写到存储器中
