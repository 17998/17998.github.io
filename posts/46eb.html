1.简介

Spring框架是由于软件开发的复杂性而创建的轻量级控制反转]（IoC）和面向切面（AOP）的容器框架。

  • 目的：解决企业应用开发的复杂性
  • 功能：使用基本的JavaBean代替EJB，并提供了更多的企业应用功能
  • 范围：任何Java应用
  • 优点：
    • 开源的免费的容器(框架)
    • 轻量级的，非入侵式的框架
    • 支持事务的处理，对框架整合的支持
  • 特性：
    • 控制反转(IOC)
    • 面向切面编程(AOP)

Spring是一个轻量级的控制反转，面向切面编程的框架

3.IOC

3.1IoC理论推导

之前

  • dao

    package cn.yunmo.dao;

public interface UserDao {
    void getUser();
}

public class UserDaoImpl implements UserDao{

    @Override

    }

public class UserDaoMySqlImpl implements UserDao{

    public void getUser() {
        System.out.println("MySql数据获取");
    }
}

  • service

    package cn.yunmo.service;

public interface UserService {
    void getUser();
}

package cn.yunmo.service;

import cn.yunmo.dao.UserDao;

public class UserServiceImpl implements UserService{
private UserDao userDao = new UserDaoImpl();

  @Override
  public void getUser() {
      userDao.getUser();
  }

}


+ test

```java
public class MyTest {

        //用户实际调用的是业务逻辑层

在我们之前的业务中，根据用户的需求会更改service中源代码

优化

public class UserServiceImpl implements UserService{
   private UserDao userDao;

    //利用set进行动态的注入
    public void setUserDao(UserDao userDao) {
        this.userDao = userDao;
    }

    @Override
    public void getUser() {
        userDao.getUser();
    }
}
public class MyTest {
    public static void main(String[] args) {
        //用户实际调用的是业务逻辑层
        UserServiceImpl userService = new UserServiceImpl();
        userService.setUserDao(new UserDaoMySqlImpl());
        userService.getUser();
    }
}
  • 之前，程序是主动创建对象，控制权在程序员手上
  • 之后，程序不再具有主动性，而是变成了被动的接受对象

3.2IOC本质

控制反转，是一种设计思想，DI(依赖注入)是实现IoC的一种方法，也有人认为DI只是IoC的另一种说法。没有IoC的程序中，我们使用面向对象编程，对象的创建与对象间的依赖关系完全硬编码在程序中，对象的创建由程序自己控制，控制反转后将对象的创建转移给第三方，所谓控制反转就是：获得依赖对象的方式反转了

控制反转是一种通过描述(xml或注解)并通过第三方去生产或获取特定对象的方式。在spring中实现控制反转的是Ioc容器，其实现方法是依赖注入

4.第一个Spring程序

  • 实体类
package cn.yunmo.pojo;

public class Hello {
    private String str;

    public String getStr() {
        return str;
    }

    public void setStr(String str) {
        this.str = str;
    }

    @Override
    public String toString() {
        return "hello{" +
                "str='" + str + '\'' +
                '}';
    }
}
  • 核心配置文件
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
       https://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--使用spring来创建对象，在spring中这些都称为bean
        id表示单个bean
        class定义bean的类型，使用完全限定的类名

        Hello hello = new Hello();
        bean=对象
        id=变量名
        class=new 的对象——》new Hello()
        property 相当于给对象中的属性设置值
    -->
    <bean id="hello" class="cn.yunmo.pojo.Hello">
        <!--
            ref：引用spring容器中已经创建好的bean
            value：基本类型赋一个具体的值
        -->
        <property name="str" value="Spring"/>
    </bean>

</beans>
  • 测试
import cn.yunmo.pojo.Hello;
import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;

public class MyTest {
    public static void main(String[] args) {
        //获取Spring的上下文对象(拿到spring的容器)
        ApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");
        //我们的对象都在Spring中管理了，我们要使用直接去里面取出来就可以了
        Hello hello = (Hello) context.getBean("hello");
        System.out.println(hello.toString());
    }
}

<bean id="user" class="cn.yunmo.pojo.User">
    <property name="name" value="杨奇"/>

通过无参函数创建对象，在通过setter()方法为属性赋值

5.2有参构造函数构造

package cn.yunmo.pojo;

public class User {
    private  String name;

    public User(String name){
        this.name = name;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
    public void show(){
        System.out.println("name="+name);
    }
}

  • 参数下标

    <bean id="user" class="cn.yunmo.pojo.User">
    <constructor-arg index="0" value="张三"/>
</bean>-->

  • 参数类型

    <bean id="user" class="cn.yunmo.pojo.User">
    <constructor-arg type="java.lang.String" value="张三"/>
</bean>

<!--引用类型必须写全限定类名-->

  • 参数名

    <bean id="user" class="cn.yunmo.pojo.User">
    <constructor-arg name="name" value="张三"/>
</bean>

总结：

在配置文件加载的时候，所有的bean只要注册到Spring容器中就会被实例化，而不是在用的时候才去实例化

当多次取用同一个bean时，这两个bean是一样的

【完善注入信息】

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">

    <bean id="student" class="cn.yunmo.pojo.Student">

7.3p命名和c命名注入

  • 实体

    package cn.yunmo.pojo;

public class User {
    private String name;
    private int age;

    //p命名注入使用无参构造函数
    public User() {

    }

    //c命名注入使用无参构造函数
    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }

  • xml

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:p="http://www.springframework.org/schema/p"
       xmlns:c="http://www.springframework.org/schema/c"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--使用p命名和c命名注入必须要引入约束-->

    <!--p命名空间注入，可以直接注入属性的值，无参构造函数 Properties-->
    <bean id="user1" class="cn.yunmo.pojo.User" p:name="杨奇" p:age="22"/>

    <!--c命名空间注入，通过构造器注入,有参构造函数  Constructor-->
    <bean id="user2" class="cn.yunmo.pojo.User" c:name="杨奇" c:age="22"/>
</beans>

    注意点：使用p命名和c命名注入必须要引入约束

9.1.3隐式的自动装配bean【重要】

一个人有两个宠物

  • 实体
package cn.yunmo.pojo;

public class Dog {
    public void shout(){
        System.out.println("wang~~~");
    }
}


package cn.yunmo.pojo;

public class Cat {
    public void shout(){
        System.out.println("miao~~~");
    }
}


package cn.yunmo.pojo;

public class People {
    private String name;
    private Cat cat;
    private Dog dog;

    public void setName(String name) {
        this.name = name;
    }


    public void setCat(Cat cat) {
        this.cat = cat;
    }

    public void setDog(Dog dog) {
        this.dog = dog;
    }

    @Override
    public String toString() {
        return "People{" +
                "name='" + name + '\'' +
                ", cat=" + cat +
                ", dog=" + dog +
                '}';
    }
}
9.1.3.1byName自动装配

byName：会自动在容器上下文中查找和自己对象的set方法后面的值【setXXX，后面的值说的是XXX】对应的bean的Id进行注入

<bean id="cat" class="cn.yunmo.pojo.Cat"/>
<bean id="dog" class="cn.yunmo.pojo.Dog"/>

<bean id="people" class="cn.yunmo.pojo.People" autowire="byName">
        <property name="name" value="yangqi"/>
</bean>
9.1.3.2byType自动装配

byType：会自动在容器上下文中查找和自己对象属性类型相同【根据class指定，如果有多个bean的class值一样则不能装配】的bean,可以不要id

<bean class="cn.yunmo.pojo.People" autowire="byName">
       <property name="name" value="yangqi"/>
</bean>

注意：

  • byName的时候，需要保证所有的bean的id唯一，并且这个bean需要和自动注入的属性的set方法的值一致
  • byType的时候，需要保证所有的bean的class唯一，并且这个bean需要和自动注入的属性的类型一致
