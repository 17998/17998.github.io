一、初始ElasticSearch

1.1 了解ES

elasticsearch是一款非常强大的开源搜索引擎，可以帮助我们从海量数据中快速找到需要的内容

elasticsearch结合Kibana,Logstash,Beats,也就是elastic stack(ELK)。被广泛应用在日志数据分析，实时监控等领域

elasticsearch是elastic stack的核心，负责存储，搜索，分析数据，Kibana是数据可视化组件，Logstash,Beats是数据抓取组件

1.2 倒排索引

1.3 核心概念

  • 文档

  • 索引

MySQL擅长事务类型操作，可以确保数据的安全性和一致性

elasticsearch擅长海量数据的搜索，分析，计算

1.4 分词器

  1. 分词器的作用
    • 创建倒排索引时对文档分词
    • 用户搜索时，对输入的内容分词
  2. IK分词器有几种模式
    • ik_smart：只能切分，粗粒度
    • ik_max_word：最细切分，细粒度
  3. IK分词器如何拓展词条？如何停用词条
    • 利用config目录下的IKAnalyzer.cfg.xml添加拓展词典和停用词典
    • 在词典中添加拓展词条和停用词条

二、索引库操作

2.1 mapping映射属性

mapping是对索引库中文档的约束，常见的mapping属性包括：

  • type：字段数据类型，常见的简单类型有：

    {
  "age":18,
  "weight":128.6,
  "email":"xxx@qq.com",
  "score":[89,68,74,91],
  "name":{
    "firstName":"三",
    "lastName":"张"
  }
}
    • 字符串
      • text：可分词的文本
      • keyword：精确值，例如品牌、国家、IP值
    • 数值
      • long、Integer、short、byte、double、float
    • 布尔
      • boolean
    • 日期
      • date
    • 对象
      • object

    es中没有数组类型，但允许某个属性有多个值

  • index：是否创建索引，默认为true

    • true：创建倒排索引，将来可以参与搜索
    • false：不创建倒排索引

  • analyzer：使用那种分词器

  • properties：该字段的子字段

三、文档操作

3.1 新增文档

POST /索引库名/_doc/id
{
    "field1":"",
    "field2":"",
     ...
}

3.2 查询文档

GET /索引库名/_doc/id

3.3 删除文档

GET /索引库名/_doc/id

3.4 修改文档

方式一：全量修改，删除旧文档，添加新文档

PUT /索引库名/_doc/id
{
    "field1":"",
    "field2":"",
     ...
}

方式二：增量修改，修改指定字段

POST /索引库名/_update/id
{
    "doc":{
        "field1":"value1",
        ...
    }
}

四、RestClient

什么是RestClient

ES官方提供了各种不同语言的客户端用来操作ES，这些客户端的本质就是用来组装DSL语句，通过http请求发送给ES
