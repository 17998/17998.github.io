1.5 Bean实例化

1.5.1 构造方法(常用)

  • 调用类中的无参构造函数来实例化bean

1.5.2 静态工厂(了解)

  • 静态工厂可以直接用类名来调用静态方法，不需要先创建工厂的bean

1.5.3 实例工厂(了解)

  • 实例工厂方法不是静态方法，需要先将工厂的bean创建出来，再用来创建需要的bean
  • 创建出来的工厂bean只是为了创建我们想要的bean，实际并没有多大意义，而且工厂方法名也不统一，所以产生了第四种实例化bean的方法

1.5.4 FactoryBean(实用)

  • FactoryBean接口是一个泛型接口，在使用时可以直接使用自己要创建的bean类型

1.7.2 构造器注入

  • 在bean中定义引用类型属性并提供可访问的构造方法
  • 引用类型

  • 简单类型

public class UserDao {
    private String name;

    private int age;

    public UserDao(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void method(){
        System.out.println("user dao method...");
        System.out.println("name:"+name+",age:"+age);
    }
}

public class UserService {
    private UserDao userDao;

    public UserService(UserDao userDao) {
        this.userDao = userDao;
    }

    public void method(){
        System.out.println("user service method... ");
        userDao.method();
    }
}
  • name的值是构造方法中形参的名字
<bean id="userDao" class="com.yunmo.dao.UserDao">
    <constructor-arg name="name" value="zhangsan"/>
    <constructor-arg name="age" value="24"/>
</bean>

<bean id="userService" class="com.yunmo.service.UserService">
    <constructor-arg name="userDao" ref="userDao"/>
</bean>

1.11 加载Properties文件

  1. 开启context命名空间

    <beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       <!--context命名空间-->
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
                           http://www.springframework.org/schema/beans/spring-beans.xsd
                            <!--context命名空间-->
                           http://www.springframework.org/schema/context
                           http://www.springframework.org/schema/context/spring-context.xsd">

  2. 使用context命名空间加载属性文件

    <context:property-placeholder location="jdbc.properties"/>

  3. 使用属性占位符读取属性文件中的属性

    <bean id="dataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource">
    <property name="driverClass" value="${jdbc.driver}"/>
    <property name="jdbcUrl" value="${jdbc.url}"/>
    <property name="user" value="${jdbc.username}"/>
    <property name="password" value="${jdbc.password}"/>
</bean>

  • 不加载系统属性(属性文件中定义的key与系统中定义的key相同时，系统中定义的key优先级要更高)

  • 加载多个Properties文件

1.12.4 BeanFactory

  • BeanFactory是容器的顶层接口，也是早期用来加载容器的接口

  • BeanFactory容器加载完成不会调用构造方法来实例化bean，而是在使用bean的时候再调用

  • ApplicationContext在容器加载完成就会调用构造方法来初始化容器中的bean，但可以在bean标签中使用lazy-init属性设置ApplicationContext接口也是懒加载

1.15 纯注解开发模式

  • @Configuration标志此类是一个配置类，代替了如下内容

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
                           http://www.springframework.org/schema/beans/spring-beans.xsd">

 
</beans>

  • @ComponentScan(“com.yunmo”)开启组件扫描，代替了配置文件中如下内容

    <context:component-scan base-package="com.yunmo"/>

  • 从此可以不再使用xml文件作为Spring的核心配置文件了
