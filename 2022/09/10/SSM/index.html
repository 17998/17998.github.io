  2. 数据源配置类

    public class DataSourceConfig {
    @Value("${jdbc.driver}")
    private String driver;
    @Value("${jdbc.url}")
    private String url;
    @Value("${jdbc.username}")
    private String username;
    @Value("${jdbc.password}")
    private String password;

    @Bean
    public DataSource getDataSource(){
        DruidDataSource dataSource = new DruidDataSource();
        dataSource.setDriverClassName(driver);
        dataSource.setUrl(url);
        dataSource.setUsername(username);
        dataSource.setPassword(password);
        return dataSource;
    }
}

  3. MyBatis配置类(不用MyBatis的xml文件了)

    public class MyBatisConfig {
    @Bean
    public SqlSessionFactoryBean sqlSessionFactoryBean(DataSource dataSource){
        SqlSessionFactoryBean ssfb = new SqlSessionFactoryBean();
        ssfb.setTypeAliasesPackage("com.yunmo.pojo");
        ssfb.setDataSource(dataSource);
        return ssfb;
    }

    @Bean
    public MapperScannerConfigurer mapperScannerConfigurer(){
        MapperScannerConfigurer msc = new MapperScannerConfigurer();
        msc.setBasePackage("com.yunmo.dao");
        return msc;
    }
}

  4. 创建Spring的核心配置类

@ComponentScan("com.yunmo")
@PropertySource("jdbc.properties")

1.23.1 AOP概念与作用

1.23.2 AOP核心概念

  • 连接点代表所有的方法
  • 切入点代表要追加功能的方法(一个或多个)
  • 代表要追加的功能(共性功能)
  • 通知类用来定义通知的类
  • 描述通知与切入点之间的关系(这个通知要加在那个切入点上)

1.23.3 AOP入门案例

  • 也可以直接将切入点写在通知上

    @Before("execution(void com.yunmo.dao.impl.BookDaoImpl.update())")

}

  • @Component是让Spring来管理，加载成bean
  • @Aspect告诉Spring把当前类当AOP来处理，不是普通bean
  • @EnableAspectJAutoProxy告诉Spring这里有用注解开发的AOP(开启AOP注解)

1.23.4 AOP工作流程

  • 没有使用的切入点不会读取

1.23.7 AOP通知获取原始方法数据

  • JointPoint joinpoint参数必须放到第一个位置

  1. 获取参数

    //通知
@Pointcut("execution(* com.yunmo.dao.BookDao.findname(..))")
private void joinpoint(){};

@Before("joinpoint()")
public void method(JoinPoint joinPoint){
    Object[] args = joinPoint.getArgs();
    System.out.println(Arrays.toString(args));
}

//原始方法
@Override
public String findname(int id) {
    System.out.println("id:"+id);
    return "云末";
}

public static void main(String[] args) {
    ApplicationContext applicationContext = new AnnotationConfigApplicationContext(SpringConfig.class);
    BookDao bookDao = applicationContext.getBean(BookDao.class);
    bookDao.findname(10);
}

  • 其他通知类型同理

  • 在环绕通知里，我们可以修改获取到的参数，然后再传给原始方法

    @Around("joinpoint()")
public void method(ProceedingJoinPoint joinPoint) throws Throwable {
    Object[] args = joinPoint.getArgs();
    System.out.println(Arrays.toString(args));
    args[0] = 20;
    Object obj = joinPoint.proceed(args);
}

  1. 获取返回值

    • returning：如果原始方法有返回值就将返回值放到ret形参中
    @AfterReturning(value = "joinpoint()",returning = "ret")
public void method(Object ret){
    System.out.println("原始方法返回值："+ret);
}

    • 原始方法返回值就是obj
    @Around("joinpoint()")
public void method(ProceedingJoinPoint joinPoint) throws Throwable{
    Object obj = joinPoint.proceed();
}

  2. 获取异常

    • 将返回的异常存到形参e中
    @AfterThrowing(value = "joinpoint()",throwing = "e")
public void method(Throwable e){
    System.out.println("原始方法抛出的异常："+e);
}

@Override
public String findname(int id) {
    System.out.println("id:"+id);
    int x = 10/0;
    return "云末";
}

    • 捕捉到异常
    @Around("joinpoint()")
public void method(ProceedingJoinPoint joinPoint){
    try {
        Object obj = joinPoint.proceed();
    } catch (Throwable throwable) {
        throwable.printStackTrace();
    }
}

1.24.3 Spring事务角色

  • 上述案例中，业务层方法transfer是事务管理员，transfer调用的两个数据访问层的方式是事务协调员

1.24.4 Spring事务属性

  • 并不是出现所有异常事务都会进行回滚，比如IOException

    //做出如下配置，当出现IOException时，就可以执行回滚
@Transactional(rollbackFor = IOException.class)

1.24.5 事务传播行为

  • transfer开启了一个事务，log也开启了一个事务，按理说log事务应该加入到transfer事务中，与transfer事务同成功同失败

  • 事务传播行为可以让事务协调员具有不同的事务行为，此处transfer调用log，log不是加入其事务，而是重新开启一个新的事务，不会与transfer事务同成功同失败

//让Tomcat服务器加载SpringMVC的配置
public class TomcatConfig extends AbstractDispatcherServletInitializer {
    //加载SpringMVC容器配置
    //Tomcat容器启动的时候就会加载这个类，然后加载SpringMVC容器

        //所有请求归属SpringMVC处理
        return new String[]{"/"};
    }
    //加载Spring容器配置
    @Override
    protected WebApplicationContext createRootApplicationContext() {
        return null;
    }
}
