二、数据类型

Redis存储的是key-value结构的数据，其中key是字符串类型，value有五种常见的数据类型

  • 字符串
  • hash
  • list
  • set
  • sort set

三、常用命令

redis的key区分大小写

key可以用冒号隔开命名，这样的好处，可以更容易理解key的含义，方便后期扩展使用

3.1 字符串string操作命令

  • 底层结构是一个HashMap

  • GETRANGE：截取某一个字符串指定范围内容，类似java中的subString方法

    127.0.0.1:6379> set name zhangsan
OK
127.0.0.1:6379> GETRANGE name 0 -1
"zhangsan"
127.0.0.1:6379> GETRANGE name 0 2
"zha"
127.0.0.1:6379> GETRANGE name -2 -1
"an"
127.0.0.1:6379> GETRANGE name 0 -2
"zhangsa"

  • GETSET key value：替换一个key中的value值

    127.0.0.1:6379>getset username admin

    将username的值设置为admin

  • MSET key value [key value …]：同时加入多个key-value

    M：multi

    127.0.0.1:6379>mset key1 val1 key2 val2

  • MGET key [key …]：同时获取多个key的value值

    127.0.0.1:6379>mget key1 key2 key3

  • DEL key1 [key2…]：删除一个或者多个key-value键值对

    127.0.0.1:6379>del username age sex

  • INCR key：根据一个key中的整数数字+1，非整数数字会报错

    127.0.0.1:6379[1]> incr age   ---> age = age+1

  • INCRBY key increment：根据一个key中的整数数字加指定整数，非整数数字会报错

    127.0.0.1:6379[1]> incrby age 10  ---> age = age+10

  • DECR key：根据一个key中的整数数字-1，非整数数字会报错

    127.0.0.1:6379[1]> decr age   ---> age = age-1

  • DECRBY key decrement：根据一个key中的整数数字减指定整数，非整数数字会报错

    127.0.0.1:6379[1]> decrby age 10

  • APPEND key value：根据一个key中的内容追加指定内容，如果key不存在，则生成一条新的k-v记录

    127.0.0.1:6379>append username world

  • STRLEN key：返回key对应值的长度，如果键不存在则返回0

    127.0.0.1:6379>strlen key

3.2 哈希 hash操作命令

  • 匹配java中的hashmap对象/ 或pojo

  • HGETALL支持key用冒号

  • HMSET key field value [field value …]：为对象set多个属性、值(一个属性对应一个值) 支持key用冒号

    127.0.0.1:6379>hmset user username zhangsan password 123456
127.0.0.1:6379>hmset user:1001 username zhangsan age 10 sex man

  • HSETNX key field value：如果对象或者对象中的属性不存在时，才会执行set对象或者set对象中的属性和值

    127.0.0.1:6379>hsetnx user username zhangsan

  • HMGET key field [field …]：获取多个字段值

    127.0.0.1:6379>hmget user username age

  • HINCRBY key field increment：增加指定的大小，返回增加后的结果值，字段的类型也必须是数值型, 否则会报错

    127.0.0.1:6379>hincrby user password 5

  • HEXISTS key field：判断一个对象或者对象中某一个字段是否存在

    127.0.0.1:6379>hexists user username

    存在返回1，不存在返回0

  • HLEN key：获取对象中的字段个数

    hlen user

3.3 列表List操作命令

  • 底层: linkedlist 双向链表(push/pop,左右都支持)
  • L：左边，R：右边

  • LPUSHX key value [value …]：如果列表存在，则向列表左边增加元素

    127.0.0.1:6379> lpushx list 3 1

  • RPUSH key value [value …]：向列表右边增加元素

    127.0.0.1:6379> rpush list 5 6 1

  • RPUSHX key value [value …]：如果列表存在，则向列表右边增加元素

    127.0.0.1:6379> rpushx list 5 6 1

    “-1”代表最后边的一个元素

  • LPOP key || RPOP key：从列表两端移除元素，POP命令从列表左边或者右边弹出一个元素

    1. 第一步是将列表左边或者右边的元素从列表中移除
    2. 第二步是返回被移除的元素值。
    127.0.0.1:6379> rpop list
127.0.0.1:6379> lpop list

  • BLPOP key || BRPOP key：和LPOP||RPOP功能类似，但是此种方式是阻塞的

    1. 返回的是一个key和value，key是集合的key, value是取出来的值
    2. 在传递参数时，多增加了一个时间，代表阻塞等待的时间
    3. 此种方式可以同时取多个list中的数据，以谁先取到为结束条件，如果都没取到，则被阻塞，直到超时，返回nil
    127.0.0.1:6379> blpop list1 list2 2
"list"
"100"
127.0.0.1:6379> blpop list1 list2 2
(nil)

  • LREM key count value：删除列表中指定的值，LREM命令会删除列表中前count个值为value的元素，返回实际删除的元素个数。根据count值的不同，该命令的执行方式会有所不同：

    127.0.0.1:6379> lrem list 3 9
    1. 当count>0时，LREM会从列表左边开始删除指定个数。
    2. 当count<0时，LREM会从列表右边开始删除指定个数。
    3. 当count=0时，LREM删除所有值为value的元素。

  • LINDEX key index：获得指定索引的元素值

    127.0.0.1:6379> lindex list 2

  • LSET key index value：设置指定索引的元素值(修改, 将原值覆盖)

    127.0.0.1:6379> lset list 2 2

  • LTRIM key start stop：只保留当前list中start-stop下标内容，其余都删除

    127.0.0.1:6379> ltrim list 0 2

  • LINSERT key BEFORE|AFTER pivot value：向列表中插入元素，该命令首先会在列表中从左到右查找值为pivot的元素，然后根据第二个参数是BEFORE还是AFTER来决定将value插入到该元素的前面还是后面

    127.0.0.1:6379> linsert list before 2 7
127.0.0.1:6379> linsert list after 3 6

  • RPOPLPUSH source destination：将最后位的一个元素从一个列表转移到另一个列表中

    将source右边的元素删除，再插入到destination的左边

    127.0.0.1:6379> rpoplpush list newlist

3.4 无序集合操作命令

  • 对标到java中的hashset，一组数据(不重复,没有位置要求)

  • SISMEMBER key member：判断元素是否在集合中

    127.0.0.1:6379> sismember set a 1

    存在返回1，不存在返回0

  • SDIFF key [key …]：集合的差集运算A-B，属于A并且不属于B的元素构成的集合。

    DIFF: different

    127.0.0.1:6379> sadd setA 1 2 3
127.0.0.1:6379> sadd setB 2 3 4
127.0.0.1:6379> sdiff setA setB
"1"
127.0.0.1:6379> sdiff setB setA
"4"

    场景: 判断非共同好友

  • SINTER key [key …]：集合的交集运算A ∩ B，属于A且属于B的元素构成的集合

    127.0.0.1:6379> sinter setA setB
"2"
"3"

    场景:判断共有的用户

  • SUNION key [key …]：集合的并集运算A ∪ B，属于A或者属于B的元素构成的集合

    127.0.0.1:6379> sunion setA setB
"1"
"2"
"3"
"4"

    场景: 本地通讯录和云备份通讯录合并

  • SPOP key [count]：从集合中弹出一个元素(或多个)，由于集合是无序的，所有SPOP命令会从集合中随机选择一个元素弹出

    场景: 网络抽奖

3.5 有序集合操作命令

场景:

  1. 热门话题排行榜(通过数据产生排序且唯一的集合)
  2. 文章的热度/话题热度 音乐/游戏
  3. 微博的热搜(搜索的次数, 每分钟更新)

  • zadd key score1 member1 [score2 member2]

    127.0.0.1:6379> zadd zset 1.0 a 2.0 b 3.0 c 0.5 d

  • ZSCORE key member：获取元素的分数

    127.0.0.1:6379> zscore zset b
2.0

  • ZREVRANGE key start stop [WITHSCORES]：按照元素分数从大到小的顺序返回索引从start到stop之间的所有元素（包含两端的元素），如果需要获得元素的分数可以在命令尾部加上WITHSCORES参数

    127.0.0.1:6379> zrevrange zset 0 2

  • ZRANK key member：从小到大，获取元素的排名

    zrank zset b

  • ZREVRANK key member：从大到小，获取元素的排名

    zrevrank zset b

  • ZRANGEBYSCORE key min max [WITHSCORES] [LIMIT offset count]：获得指定分数范围的元素

    Limit 分页显示,从offset位置开始 显示count条记录

    127.0.0.1:6379> zrangebyscore zset 1 3
"a"
"b"
"c"

  • ZCARD key：获得集合中元素的数量

    127.0.0.1:6379> zcard zset
(integer) 4

  • ZCOUNT key min max：获得指定分数范围内的元素个数

    127.0.0.1:6379> zcount zset 1 3
(integer) 3

  • ZREMRANGEBYRANK key start stop：按照排名范围删除元素

    127.0.0.1:6379> zremrangebyrank zset 1 3
(integer) 3

  • ZREMRANGEBYSCORE key min max：按照分数范围删除元素

    127.0.0.1:6379> ZREMRANGEBYSCORE scoreboard 1 3

3.6 通用操作命令

  • dbsize：查看当前仓库中存在的key总共多少个

    192.168.3.136:6379> dbsize
(integer) 5

  • select number：来切换数据仓库

  • rename：重命名key

    127.0.0.1:6379> rename list2 list1

    将list2重命名为list1

  • flushall：清除所有缓存

3.7 设置生存时间

  • EXPIRE key seconds：设置key的生存时间（单位：秒）key在多少秒后会自动删除
  • PEXPIRE key milliseconds：生存时间设置单位为：毫秒
  • PTTL key：查看key剩余的生存时间(单位为毫秒)
  • PERSIST key：清除生存时间(长期存在)

四、Redis服务配置

4.1 Redis解决持久化

  • 在linux中安装了redis以后，默认是没有dump.rdb文件的，而这个文件就是用来做磁盘本地持久化处理的，所以需要手动添加这个文件，可以从解压缩的文件中直接拷贝一个到/usr/local/bin目录(配置文件在该目录下)，然后再启动redis就可以正常持久化了

    • 900秒，至少有一个key的值更新就持久化
    • 300秒，至少有10个key的值更新就持久化
    • 60秒，至少有一个key的值更新就持久化

4.2 外部访问Redis服务

  • 如果需要让外部访问到Linux中的redis服务，必须修改redis.conf配置文件两个位置，如果bin目录下没有配置文件, 就需要从压缩包里拷贝一份过去

  • cp /usr/java/redis-4.0.0/redis.conf /usr/local/bin/redis.conf

    1. 允许远程：bind 127.0.0.1 注释掉
    2. 添加权限：# requirepass foobared 打开注释，并且把foobared修改为自己想要的密码

  • 启动服务时，需要加上对应的conf配置文件，才能生效

    ./redis-server ./redis.conf

  • 客户端远程启动时，需要添加对应的密码，两种操作模式：

    1. 第一种，连接的时候就输入密码

      ./redis-cli -h 192.168.3.100 -p 6379 -a 密码

# -h: host 服务器地址
# -p: port 端口号
# -a: auth 权限

    2. 分两步，先连接，再通过auth输入密码

      # 先连接上
./redis-cli -h 192.168.3.100 -p 6379
# 再输入密码
auth 密码

4.3 开启Redis后台服务

  • 在redis.conf文件中，找到daemonize no，把no改为 yes即可

5.2 Jedis

  • 操作都和命令相对应
public void testJedis(){
    //1. 获取连接
    //Jedis jedis = new Jedis("localhost",6379);
    JedisPool jedisPool = new JedisPool("127.0.0.0",6379);
    Jedis jedis = jedisPool.getResource();
    jedis.auth("123456");

    //2. 执行操作
    jedis.set("username","yangqi");

    String name = jedis.get("username");
    System.out.println(name);

    jedis.del("username");

    //3. 关闭连接
    jedis.close();
}

5.3 Spring Data Redis

5.4 SpringBoot中集成Redis

  1. 导入依赖

    <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
<!--提供redis连接池-->
<dependency>
    <groupId>redis.clients</groupId>
    <artifactId>jedis</artifactId>
</dependency>

  2. 配置文件

    spring:
  redis:
    host: 127.0.0.1
    port: 6379
    password: '@Yq811125'
    database: 0
    # jedis连接池配置
    jedis:
      pool:
        max-active: 8
        max-idle: 8
        min-idle: 0

  3. 创建Redis配置类

    @Configuration
public class RedisConfig {
    @Bean
    public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory redisConnectionFactory) {
        RedisTemplate<String, Object> redisTemplate = new RedisTemplate<>();
        //配置连接工厂
        redisTemplate.setConnectionFactory(redisConnectionFactory);
        //jackson序列化器
        Jackson2JsonRedisSerializer<Object> jsonRedisSerializer = new Jackson2JsonRedisSerializer(Object.class);
        ObjectMapper objectMapper = new ObjectMapper();
        objectMapper.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
        objectMapper.enableDefaultTyping(ObjectMapper.DefaultTyping.NON_FINAL);
        jsonRedisSerializer.setObjectMapper(objectMapper);

        //String序列化器
        StringRedisSerializer stringRedisSerializer = new StringRedisSerializer();

        redisTemplate.setKeySerializer(stringRedisSerializer);
        redisTemplate.setHashKeySerializer(stringRedisSerializer);
        redisTemplate.setValueSerializer(jsonRedisSerializer);
        redisTemplate.setHashValueSerializer(jsonRedisSerializer);

        redisTemplate.afterPropertiesSet();
        return redisTemplate;
    }
}
