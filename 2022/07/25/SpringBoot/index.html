2.7 定时器

  • SpringBoot提供了定时器

  • 使用方式

    1. 在启动类上添加@EnableScheduling，开启定时任务
    2. 在方法上添加@Scheduled(initialDelay = 10000,fixedDelay=1000)，表示这是一个定时器方法
    @Component
public class MyTimer {
    @Autowired
    private UserService userService;

    /**
     * 默认的定时任务是单线程任务，如果有多个定时任务，并且其中某个定时任务执行时间很长，会阻塞其他定时任务的执行
     * 我们可以开启多线程来执行多个定时任务，避免发生阻塞
     */
    @Scheduled(initialDelay = 10000,fixedDelay=1000)
    public void selectUserNameIsNull(){
        System.out.println(Thread.currentThread().getName());
        LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<>();
        lqw.isNull(User::getUsername);
        lqw.or().eq(User::getUsername,"");
        List<User> users = userService.list(lqw);
        users.forEach(System.out::println);
    }
}
    • 首先定时器方法所在的类要被Spring管理

  • 默认是单线程执行定时任务，通过如下方式来创建多线程池来执行定时任务，避免任务发生阻塞

  • 方式一：

    1. 在启动类上添加注解@EnableAsync，开启异步执行

    2. 创建一个多线程池的bean，交给Spring来管理

      @Bean
public Executor executor(){
    ThreadPoolExecutor executor = new ThreadPoolExecutor(5,10,5000, TimeUnit.MILLISECONDS,
                                                         new LinkedBlockingDeque<>(10),
                                                         new ThreadPoolExecutor.DiscardPolicy());
    return executor;
}

    3. 在定时方法上添加注解@Async(“executor”)，value值是多线程池的bean的名字

  • 方式二：

    1. 实现SchedulingConfigurer，重写其中的configureTasks方法，重新定义线程池对

      @Configuration
public class ScheduleConfig implements SchedulingConfigurer {
    @Override
    public void configureTasks(ScheduledTaskRegistrar scheduledTaskRegistrar) {
        /**
         * 默认线程池是 Executors.newSingleThreadScheduledExecutor()
         */
        scheduledTaskRegistrar.setScheduler(Executors.newScheduledThreadPool(5));
    }
}
