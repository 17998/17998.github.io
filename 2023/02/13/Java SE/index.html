Java SE

day01

1.1 java语言特性

  1. 简单性
    • Java很简单，和C++相比，Java是单继承的，用接口代替了类的多继承，摒弃了C++指针概念，自动垃圾回收机制，提供了大量API
  2. 安全性
    • Java语言是安全可靠的
  3. 健壮性
    • 程序稳定，Java有垃圾回收器，当内存中垃圾产生到一定数量，垃圾回收器会自动回收垃圾，不需要程序员手动调用，C++没有垃圾回收器
  4. 面向对象性
    • 相对于面向过程，调用某个实体的对象，通过对象自身的方法来完成功能
  5. 多线程性
    • 提高效率
  6. 可移植性
    • Java的类库提供了针对不同平台的接口，所以这些类库可以被移植
    • Java程序无论运行在何种CPU，操作系统，何种编译器，都将产生同样的结果
  7. 跨平台性
    • Java代码可以一次编译到处运行(Java代码可以在不同的系统中运行,比如:Windows,Linux,Android,Mac)

1.2 java跨平台原理(重点)

  • 所有的Java源代码都是先编译再运行

    • 编译的作用：检查Java源码中的语法，语法没有错误就会生成.class文件
    • 运行的作用：输出结果

  • 由于不同的系统安装了对应的虚拟机JVM，Java代码可以一次编译到处运行

1.3 什么是面向对象

  1. 面向对象是一种编程思想
  2. 面向过程思想更加注重这个事情的每一个步骤以及顺序，面向对象它更加注重事情有哪些参与者，需求里面有哪些对象，这些对象各自需要做些什么事情，将其拆解成一个个模块和对象，这样会更易于维护和拓展

1.4 JDK，JRE，JVM之间的关系(笔试题，简答题，重点)

  • JDK: Java开发工具包 Java Development Kit 是写Java代码之前必须安装的工具
  • JRE: Java运行时环境 Java Runtime Environment 是Java运行时必须依赖环境，为Java代码的运行提供了核心类库的支持
  • JVM: Java虚拟机 Java Virtual Machine，JVM就是一个软件，JVM针对不同的操作系统提供了不同版本的JVM
  • 关系：JDK包含JRE，JRE包含JVM

1.5 为什么配置环境变量

  • 开发Java程序，需要使用JDK提供的开发工具（比如javac.exe编译命令、java.exe运行命令），这些工具在JDK的安装目录的bin目录下，如果不配置环境变量，那么这些命令只可以在该目录下执行。我们不可能把所有的java文件都放到JDK的bin目录下
  • 所以配置环境变量的作用就是可以使bin目录下的java相关命令可以在任意目录下使用

1.6 不同操作系统Path路径分隔符

  • Windows：用分号(;)分隔
  • Linux：用冒号(:)分隔

day02

2.1 标识符命名规范

  1. 见名之意
  2. 数字，字母，下划线，$命名，并且不能以数字开头
  3. 不能用关键字命名(String不是关键字)
  4. 可以用中文命名，但不建议

  • 类名/接口名：大驼峰命名法–每个单词的首字母大写

  • 变量名/方法名：小驼峰命名法–第一个单词全部小写，后面每个单词的首字母大写

  • 常量名：每个单词所有字母全部大写

2.2 成员变量与局部变量的区别

  • 成员变量：定义在方法外，类里面的变量，不需要赋值，有默认值
  • 局部变量：定义在方法体或代码块中的变量，需要赋值，否则无法使用

2.3 数据类型(重点 取值范围 占用字节)

  • 基本数据类型（4类8种–都是关键字）
    1. 整型（占用字节数不受操作系统的影响）
      • byte
        • 取值范围：-128127（-2^72^7-1)
        • 内存：1字节
      • short
        • 取值范围：-3276832767（-2^152^15-1)
        • 内存：2个字节
      • int
        • 取值范围：-21474836482147483647(-2^312^31-1)
        • 内存：4个字节
      • long
        • 取值范围：-2^63~2^63-1
        • 规范：结尾加上L/l
        • 内存：8个字节
      • 默认值：0
      • 整型常量表示形式
        1. 十进制
        2. 八进制
        3. 十六进制
    2. 浮点型
      • float
        • 规范：结尾加上F/f
        • 内存：4个字节
      • double
        • 规范：结尾可以加上也可以不加D/d
        • 内存：8个字节
      • 默认值：0.0
    3. 字符型
      • char
      • 取值范围：065535(02^16-1)
      • 内存：2个字节
      • 默认值：空格
      • 必须有内容
    4. bool型
      • boolean
      • 内存：1个字节
      • 默认值：false

3.2 方法的定义和调用

方法：就是将一个功能抽取出来，把代码单独定义在一个大括号内，形成一个单独的功能。当我们需要这个功能的时候，就可以去调用。这样即实现了代码的复用性，也解决了代码冗余的现象。

  • 注意事项
    1. 方法必须定义在一个类中，方法之外
    2. 方法与方法之间相互独立
    3. 方法返回值和方法返回类型要一致

位运算是直接对二进制补码进行运算。(了解)

3.4 常量和变量的运算

public static void main(String[] args){
    byte b1=1;
    byte b2=2;
    byte b3=1 + 2;
    byte b4=b2 + b3;
    System.out.println(b3);
    System.out.println(b4);
}

分析： b3 = 1 + 2 ， 1 和 2 是常量，为固定不变的数据，在编译的时候（编译器javac），已经确定了 1+2 的结果并没有超过byte类型的取值范围，可以赋值给变量 b3 ，因此 b3=1 + 2 是正确的。
反之， b4 = b2 + b3 ， b2 和 b3 是变量，变量的值是可能变化的，在编译的时候，编译器javac不确定b2+b3的结果是什么，因此会将结果以int类型进行处理，所以int类型不能赋值给byte类型，因此编译失败。

3.5 笔试题

3.5.1 ++i和i++的区别

  • ++(–)在变量前面
    • 先运算再使用
  • ++(–)在变量后面
    • 先使用再运算

3.5.2 && 与 & 和 || 与 |区别

  • 当&&左边的表达式结果为false，右边的表达式不再执行
  • 当||左边的表达式结果为true，右边的表达式不再执行

day04

4.1 判断语句

  • 单分支if

    if(条件表达式){
    语句块;
}
    • 如果条件表达式为true，执行语句，否则跳过该分支，执行下面的代码
    • 如果语句块是一条语句，不是指一行代码，可以不写{}，但不建议

  • 双分支if

    if(条件表达式){
    语句块1;
}else{
    语句块2;
}
    • 条件表达式为true，执行语句块1，否则执行语句块2
    • 如果语句块是一条语句，不是指一行代码，可以不写{}，但不建议

  • 多分支if

    if(条件表达式1){
    语句块1;
}else if(条件表达式2){
    语句块2;
}else if(条件表达式3){
    语句块3;
}
...
    
else if(条件表达式n-1){
    语句块n-1;
}else{
    语句块n;
}

  • switch语句

    switch(算数表达式){
      case 常量表达式1 : 
         语句块1
         break;
      case 常量表达式2 :
         语句块2
         break;
      case 常量表达式3 : 
         语句块3
         break;
      default:
         语句块4
         break;
}

    • case后面的常量表达式结果是同种数据类型

    • 算术表达式结果类型支持如下

      • 在JDK7之前只能是int类型，也可以是byte，short，char，因为可以进行自动类型转换
      • 在JDK7及之后，可以是byte，short，char，int，String，enum

    • 当default在case之前时

      • 如果能匹配上case

        public static void main(String[] args) {
    int week = 2;
    switch (week) {
        case 4:
            System.out.println("周一");
        case 5:
            System.out.println("周二");
            break;
        default:
            System.out.println("周末");
        case 2:
            System.out.println("周三");
        case 3:
            System.out.println("周四");
            break;
    }
}

      • 如果不能匹配上case

        public static void main(String[] args) {
    int week = 1;
    switch (week) {
        case 4:
            System.out.println("周一");
        case 5:
            System.out.println("周二");
            break;
        default:
            System.out.println("周天");
        case 2:
            System.out.println("周三");
        case 3:
            System.out.println("周四");
            break;
    }
}
