day01

1.1 java语言特性

  1. 简单性
    • Java很简单，和C++相比，Java是单继承的，用接口代替了类的多继承，摒弃了C++指针概念，自动垃圾回收机制，提供了大量API
  2. 安全性
    • Java语言是安全可靠的
  3. 健壮性
    • 程序稳定，Java有垃圾回收器，当内存中垃圾产生到一定数量，垃圾回收器会自动回收垃圾，不需要程序员手动调用，C++没有垃圾回收器
  4. 面向对象性
    • 相对于面向过程，调用某个实体的对象，通过对象自身的方法来完成功能
  5. 多线程性
    • 提高效率
  6. 可移植性
    • Java的类库提供了针对不同平台的接口，所以这些类库可以被移植
    • Java程序无论运行在何种CPU，操作系统，何种编译器，都将产生同样的结果
  7. 跨平台性
    • Java代码可以一次编译到处运行(Java代码可以在不同的系统中运行,比如:Windows,Linux,Android,Mac)

1.2 java跨平台原理(重点)

  • 所有的Java源代码都是先编译再运行

    • 编译的作用：检查Java源码中的语法，语法没有错误就会生成.class文件
    • 运行的作用：输出结果

  • 由于不同的系统安装了对应的虚拟机JVM，Java代码可以一次编译到处运行

1.3 什么是面向对象

  1. 面向对象是一种编程思想
  2. 面向过程思想更加注重这个事情的每一个步骤以及顺序，面向对象它更加注重事情有哪些参与者，需求里面有哪些对象，这些对象各自需要做些什么事情，将其拆解成一个个模块和对象，这样会更易于维护和拓展

1.4 JDK，JRE，JVM之间的关系(笔试题，简答题，重点)

  • JDK: Java开发工具包 Java Development Kit 是写Java代码之前必须安装的工具
  • JRE: Java运行时环境 Java Runtime Environment 是Java运行时必须依赖环境，为Java代码的运行提供了核心类库的支持
  • JVM: Java虚拟机 Java Virtual Machine，JVM就是一个软件，JVM针对不同的操作系统提供了不同版本的JVM
  • 关系：JDK包含JRE，JRE包含JVM

1.5 为什么配置环境变量

  • 开发Java程序，需要使用JDK提供的开发工具（比如javac.exe编译命令、java.exe运行命令），这些工具在JDK的安装目录的bin目录下，如果不配置环境变量，那么这些命令只可以在该目录下执行。我们不可能把所有的java文件都放到JDK的bin目录下
  • 所以配置环境变量的作用就是可以使bin目录下的java相关命令可以在任意目录下使用

  • Linux：用冒号(:)分隔

3.4 常量和变量的运算

public static void main(String[] args){
    byte b1=1;
    byte b2=2;
    byte b3=1 + 2;
    byte b4=b2 + b3;
    System.out.println(b3);
    System.out.println(b4);
}

分析： b3 = 1 + 2 ， 1 和 2 是常量，为固定不变的数据，在编译的时候（编译器javac），已经确定了 1+2 的结果并没有超过byte类型的取值范围，可以赋值给变量 b3 ，因此 b3=1 + 2 是正确的。
反之， b4 = b2 + b3 ， b2 和 b3 是变量，变量的值是可能变化的，在编译的时候，编译器javac不确定b2+b3的结果是什么，因此会将结果以int类型进行处理，所以int类型不能赋值给byte类型，因此编译失败。

6.1 冒泡排序

public class BubbleSort {
    public static void main(String[] args) {
        int[] arr = {89,75,50,65,40};
        bubbleSort(arr);
        System.out.println(Arrays.toString(arr));
    }

    /**
     * 冒泡排序
     * @param nums 待排序的数组
     */
    public static void bubbleSort(int[] nums){
        int len = nums.length;
        if(nums == null || len<2)
            return;
        for (int i = 0; i < len-1; i++) {
            boolean flag = true;
            for(int j = 0; j < len-i-1; j++){
                if(nums[j] > nums[j+1]){
                    int temp = nums[j];
                    nums[j] = nums[j+1];
                    nums[j+1] = temp;
                    flag = false;
                }
            }
            if(flag){
                break;
            }
        }
    }
}

6.2 选择排序

public class SelectSort {
    public static void main(String[] args) {
        int[] arr = {89,75,50,65,40};
        selectSort(arr);
        System.out.println(Arrays.toString(arr));
    }
    
    public static void selectSort(int[] arr){
        int len = arr.length;
        if(arr == null || len<2)
            return;
        for(int i=0;i<len;i++){
            int index = i;
            for (int j = i+1; j < len; j++) {
                if(arr[j] < arr[index]){
                    index = j;
                }
            }
            if(index != i)
                swap(arr,i,index);
        }
    }

    public static void swap(int[] arr,int i,int j){
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}

6.3 插入排序

public class InsertSort {
    public static void main(String[] args) {
        int[] arr = {2,1,3,4,5};
        insertSort(arr);
        System.out.println(Arrays.toString(arr));
    }

    public static void insertSort(int[] arr){
        int len = arr.length;
        if(arr == null || len<2){
            return;
        }
        for (int i = 1; i < len; i++) {
            int cur = arr[i];
            for (int j = i-1; j >= 0; j--) {
                //超出数组底
                if(j==0 && cur<arr[0]){
                    for (int k = i-1; k >= 0; k--) {
                        arr[k+1] = arr[k];
                    }
                    arr[0] = cur;
                }
                if(cur < arr[j] && cur > arr[j-1]){
                    for (int k = i-1; k >= j; k--) {
                        arr[k+1] = arr[k];
                    }
                    arr[j] = cur;
                }
            }
        }
    }
}

6.4 折半查找

public class BinarySearch {
    public static void main(String[] args) {
        int[] arr = {-45,-35,-20,-10,1,20,23,34,45,50,56};
        int index = search(arr, 34);
        System.out.println(index);
    }

    public static int search(int[] arr,int target){
        int start = 0;
        int end = arr.length-1;
        while (start<=end){
            int mid = (start+end)/2;
            if(target < arr[mid] )
                end = mid-1;
            else if(target > arr[mid]){
                start = mid+1;
            }else{
                return mid;
            }
        }
        return -1;
    }

8.1 成员变量和局部变量的区别

  • 成员变量

    • 类变量：用static修饰
    • 实例变量：没有用static修饰

  • 在类中的位置不同

    • 成员变量:类中方法外
    • 局部变量:在方法中定义的变量(方法里面，方法声明)

  • 作用范围不同

    • 成员变量:类中都可以使用
    • 局部变量:只有在定义该变量的方法中使用

  • 在内存的位置不同

    • 成员变量:在堆内存
    • 局部变量:在栈内存

  • 初始值不同

    • 成员变量:有默认初始化
    • 局部变量:没有默认的初始化，必须定义赋值后在使用

  • 生命周期不同

    • 成员变量:随着对象的创建而存在，随着对象的销毁而销毁
    • 局部变量:随着方法的调用而存在，随着方法的调用完毕而销毁

如果在方法中遇到了局部变量和成员变量同名，如果直接使用变量名他是就近原则,就是局部变量

当方法中只能操作成员变量的时候 系统会自动加上this

  • 注意事项：

    2. 子类在重写父类的时候 权限不能更低
    3. 重写的时候方法声明一模一样
    4. 子类在重写父类方法时，返回值不能更大

10.3 抽象类

1.抽象类不能创建对象，如果创建，编译无法通过而报错。只能创建其非抽象子类的对象。
    理解：假设创建了抽象类的对象，调用抽象的方法，而抽象方法没有具体的方法体，没有意义。
2.抽象类中，可以有构造方法，是供子类创建对象时，初始化父类成员使用的。
    理解：子类的构造方法中，有默认的super()，需要访问父类构造方法。
3.抽象类中，不一定包含抽象方法，但是有抽象方法的类必定是抽象类。
    理解：未包含抽象方法的抽象类，目的就是不想让调用者创建该类对象，通常用于某些特殊的类结构设计。
4.抽象类的子类，必须重写抽象父类中所有的抽象方法，否则，编译无法通过而报错。除非该子类也是抽象类。
    理解：假设不重写所有抽象方法，则类中可能包含抽象方法。那么创建对象后，调用抽象的方法，没有意义。

  3. 抽象类不能实例化 因为他不是具体的
  4. 抽象类的子类
    • 如果不想覆盖抽象方法，那么该类必须是一个抽象类
    • 重写所有的抽象方法,这个类就是一个具体类
  5. 抽象类的实例化是靠具体的子类实现的，是多态的形式

10.4 接口

  • 接口中除了定义的私有方法外，其他一切常量，方法都用public修饰

10.4.1 接口概述

//定义动物训练的接口
interface AnimalTrain{
    //抽象方法
    public abstract void jump();
}

//抽象类实现接口  那抽象方法可以不用重写
abstract class Dog implements AnimalTrain{
    
}

//具体类

    @Override
    public void jump() {

public class Demo01 {
    public static void main(String[] args) {
        //接口不能new
        //但是可以提供多态的方式来创建
        //向上转型
        AnimalTrain animalTrain=new Cat();
    }
}
  • 接口是一个特殊的抽象类
  • 接口不能实例化
  • 接口的子类
    • 抽象类：不用覆写接口中的抽象方法
    • 具体类：覆写接口中的所有抽象方法

10.4.2 接口的特点

  • 接口的成员特点:
    • 构造方法: 接口没有构造方法
    • 成员变量: 只能是静态的常量，默认加上修饰符 public static final
    • 成员方法: 必须是抽象方法 ，public abstract关键字可以不写，自动生成
    • 接口的变量是静态常量，所以不能修改
public interface Person {
    int a = 10;  //默认是常量，必须初始化
    
    public static final int b = 20;
    
    void f();
    
    public abstract void g();
}
  • public static final是可以省略的
  • public abstract是可以省略的

10.4.3 接口的关系

  • 类与类: (单继承)

    • 继承关系 ：单继承，但是可以多层(级)继承，多重继承

  • 类与接口:(多实现)

    • 实现关系: 可以单实现，也可以多实现
    • 可以在继承一个类的基础上实现多个接口

  • 接口与接口关系:(多继承)

    • 继承关系，可以单继承，也可以多继承

10.4.4 抽象类和接口的区别

抽象类:
       成员变量:  可以是常量，也可以是变量
       构造方法:  有
       成员方法:  可以是抽象的，也可以是非抽象的
接口:
       成员变量:  只能是静态的常量
       成员方法： 只能是抽象方法
       构造方法： 无
       
       类与类: 继承关系  单继承，但是可以多层继承
       类与接口: 实现关系 可以单实现，也可以多实现
       接口与接口: 继承关系，可以单继承，也可以多继承

10.4.5 默认方法(面试–JDK1.8)

  • 默认default修饰，不可以省略，供子类调用或者重写

  • 默认方法有方法体，如果不加default会认为是一个抽象方法，但不应该有方法体

  • 子类重写父类的默认方法时，不需要加default

public interface Person {

10.5.2 多态的特点

    public static void function() {

    }

    }

  • 成员变量: 编译看左边,运行看左边

  • 静态方法: 编译看左边,运行看左边 ，算不上重写，访问的就是父类

10.5.3 多态的优点

  • 提高了代码的扩展性(多态保证)

10.9 笔试题

10.9.1 this()和super()在代码块中的位置

  • super()与this()都必须放在有效代码的第一行
  • super()与this()不能同时出现

10.9.2 抽象方法的注意事项

  1. 抽象类不能创建对象
  2. 抽象类中，可以有构造方法，是供子类创建对象时，初始化父类成员使用的。
  3. 抽象类中，不一定包含抽象方法，但是有抽象方法的类必定是抽象类。
  4. 抽象类的子类，必须重写抽象父类中所有的抽象方法，否则，编译无法通过而报错。除非该子类也是抽象类。

10.9.3 静态代码块,构造代码块，构造方法

  • static修饰的内容是随着类的加载而加载，并且是最先加载的，而且只加载一次
  • 每次执行构造方法之前会先执行构造代码块
  • 先调用父类的无参构造方法 再调用子类的无参构造方法
  • 父优先于子

  • 可以修饰类，变量，方法
  • final修饰类，类就不能被继承
  • final修饰方法，方法就不能被重写
